# Acceptance Gates (Stabilization)

هذه المعايير تُستخدم كـ **بوابات نجاح/فشل** قبل البدء في ميزات كبيرة، وخصوصًا بعد أي تغييرات على الإقلاع، التحميل الديناميكي، i18n، أو PTW.

## Gate 1: شاشة الدخول (Login Screen)
- **PASS**: `#login-screen` يظهر بشكل طبيعي عند عدم وجود جلسة فعالة.
- **PASS**: الضغط على تسجيل الدخول ينقل إلى التطبيق أو يظهر خطأ واضح للمستخدم (بدون تجمّد).
- **FAIL**: شاشة بيضاء/تعليق، أو خطأ JavaScript قاتل يمنع ظهور أي شاشة.

## Gate 2: تحميل الوحدات (Modules Loading)
- **PASS**: تحميل الوحدات لا ينهار بسبب `ReferenceError/TypeError`.
- **PASS**: استدعاء تحميل الوحدات يتم **مرة واحدة** (لا race يؤدي لتعليق أو تحميل مكرر).
- **PASS**: في حالة فشل تحميل موديول واحد، التطبيق يستمر ويعطي تحذير واضح (fail-open) بدل التوقف الكامل.
- **FAIL**: تعليق في bootstrap أو تكرار تحميل يسبب تباطؤ شديد/تجمد.

## Gate 3: i18n / اللغة
- **PASS**: `window.AppI18n` و/أو `window.I18n` جاهزين قبل اعتماد الوحدات عليها.
- **PASS**: تغيير اللغة لا يعيد تهيئة التطبيق بالكامل ولا يسبب loop.
- **FAIL**: تغيير اللغة يسبب استدعاءات متداخلة تؤدي لتعليق أو كسر واجهة.

## Gate 4: PTW (أهم وحدة)
- **PASS**: فتح PTW يعرض الهيكل الأساسي فورًا ثم يحمل البيانات لاحقًا بدون تعليق.
- **PASS**: حدث `language-changed` أثناء تحميل PTW لا يسبب تحميل متداخل؛ يتم عمل reload واحد بعد اكتمال التحميل.
- **PASS**: أخطاء التحميل من Backend يتم التعامل معها بإظهار واجهة fallback + زر retry بدون crash.
- **FAIL**: فتح PTW يسبب hang أو تكرار `load()` بلا نهاية أو أخطاء قاتلة.

## Gate 5: Supabase الاتصال + المزامنة
- **PASS**: `SAAS_CONFIG.useSupabaseBackend === true` و`SaaSAdapter` جاهز.
- **PASS**: قراءة بيانات أساسية تملأ `AppState.appData` بعد login/sync عبر RPC.
- **PASS**: الكتابة (save/append) ترجع نجاح أو رسالة خطأ واضحة؛ لا يوجد “نجاح صامت” مع فشل فعلي.
- **FAIL**: فشل متكرر في RPC أو أخطاء شبكة بدون fallback واضح.

## Gate 6: Local storage + Pending sync
- **PASS**: بعد Refresh، بيانات `hse_app_data` تعيد بناء الواجهة بشكل صحيح.
- **PASS**: في وضع Offline ثم Online، `pending sync` يفلش ويثبت البيانات في Supabase.
- **FAIL**: فقد بيانات بسبب overwrite غير مقصود أو coalescing خاطئ للـ queue.

## خطوات اختبار مختصرة (Smoke Test)
- مسح localStorage → فتح التطبيق → تحقق من Login.
- Login → فتح Dashboard → فتح PTW.
- تغيير اللغة → تأكد PTW/الواجهة لا تتجمد.
- إجراء حفظ بسيط → Refresh → تأكد البيانات ما زالت موجودة.
